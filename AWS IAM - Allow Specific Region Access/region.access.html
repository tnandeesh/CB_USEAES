AWS IAM - Allow Specific Region Access

IAM grants precise AWS access control. Set region-specific permissions using policies, ensuring secure resource interaction within designated areas for enhanced security and compliance

AWS Identity and Access Management (IAM) allows you to control access to AWS services and resources. In the context of specific region access, IAM enables you to define fine-grained permissions for users, groups, or roles while considering the geographical scope. By using IAM policies with conditions such as "aws:RequestedRegion," you can restrict or allow access to AWS resources based on the specified region. This level of control enhances security, compliance, and governance, ensuring that users only interact with resources in designated regions.

Step 1: Create IAM User

  1. Open the IAM Console and Navigate to "Users".
  2. Click "Create user."
  3. Enter a username and choose "Provide User access to Management console".
  4. Choose a console password (Select AutoGenerated or Custom Password).
  5. Click "Next."
  6. Skip the permissions step (do not attach any policies).
  7. Review and create the user.
  • Save the IAM User Credentials (or) Using Credentials Login to IAM User.

    • Note: This will create an IAM user without any attached policies, meaning the user has no permissions by default.

    Step 2: Create a Custom IAM Policy for IAM User

    1. Open the AWS Management Console and navigate to the IAM service (In Root User).
    2. In the left navigation pane, click on "Policies."
    3. Click on the "Create policy" button.
    4. Choose the "JSON" tab and paste the JSON policy document:
      5. { "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": [ "ec2:*", "rds:*", "s3:*" ], "Resource": "*", "Condition": { "StringEquals": { "aws:RequestedRegion": "us-east-1" } } } ] }

      Note: The provided IAM policy JSON document grants specific permissions for the actions related to EC2, RDS, and S3 services, but only in the "us-east-1" region.

      (OR)

      { "Version": "2012-10-17", "Statement": [ { "Effect": "Allow", "Action": [ "*" ], "Resource": "*", "Condition": { "StringEquals": { "aws:RequestedRegion": "us-east-1" } } } ] }

      Note: IAM Policies for Only NV-Region For All Services

    5. Click "Next."
    6. Name the policy and provide a description, Optionally.
    7. Click "Create policy."

    Step 3: Attach the Policy to the IAM User

    1. In the IAM console, click on "Users" in the left navigation pane.
    2. Select the IAM user to which you Created.
    3. Click on the "Permissions" tab.
    4. Click on "Add permission".
    5. Click on "Attach policies directly."
    6. Search for Policy which you created with name and select the custom policy you created.
    7. Click "Next".
    8. Review and Click "Add permission".

    Step 5: Test Access

    1. Sign in to the AWS Management Console using the IAM user's credentials.
    2. Attempt to perform actions in the specified regions (e.g., create an EC2 instance) to ensure that access is allowed only in the allowed regions.
