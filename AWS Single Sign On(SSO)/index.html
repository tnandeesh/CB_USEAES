AWS Single Sign On(SSO)

Single Sign-On (SSO) is an authentication process that allows users to access multiple applications with a single set of login credentials, streamlining the user experience and improving security by reducing the need for multiple passwords.

AWS Switch Role simplifies user access to different AWS accounts, allowing seamless role assumption across accounts. It streamlines user management and access control with a centralized approach. Secure role switching and a unified dashboard for administrators ensure efficient access management. Integration with identity providers, such as Active Directory, enhances authentication simplicity and security.

Step 1: Login to Account-A & Account-B as root user

  • Account-A is referred to as the Management/Organization/IAM Account.
  • Account-B is designated as the Dev Environment Account.

Step 2: Go to Account-A and Access AWS Organization

  1. Click "Create Organization".
  2. Click "Add an AWS account".
  3. Select "Invite an existing AWS account".
  4. Add Account-B Account ID and Click "Send invitation".
  5. Go to Account-B and Access AWS Organization.
  6. Click on the invitation and "Accept invitation".

Step 3: Create Single Sign-On User (IAM User)

  1. Navigate to the IAM dashboard and click on IAM Identity Center.
  2. Click on "Users" and then "Add user."
  3. Enter a username for the IAM user.
  4. Choose "Generate a one-time password that you can share with this user."
  5. Enter the user's email and provide their first and last name.
  6. Click on "Next."
  7. Optionally, add the user to a group and click on "Next."
  8. Review the user details and click on "Add User."

Using Credentials Login to the AWS Portal:

  1. Access the portal URL.
  2. Provide the IAM username.
  3. Use the one-time password generated during User creation.
  4. Reset the password as needed.

Post-Login Configuration:

  • Now, if you see that there are no applications available, ensure the setup is correct and troubleshoot any potential issues.
  • This ensures a smooth Single Sign-On experience for the user.

Step 4: Assign User Permissions

  1. In the IAM Identity Center, go to AWS Accounts.
  2. Select the desired AWS account and click on "Assign User Or Group."
  3. Choose the user you created and click on "Next."
  4. Select permissions, such as "Administrator access" and "Power user access."
  5. Click on "Next."
  6. Review and submit the assignment to the account name.
  7. Click on "Submit."

Review and Access:

  1. After the assignment is complete, log in to the AWS portal.
  2. Select the AWS account, and you should see your account listed.
  3. Click on Account and Select "Management Console."

Accessing the Account:

You are now logged into the account. You can use various AWS services.

Note: Due to the attached permissions ("Administrator access" and "Power user access"), you have the necessary privileges.
